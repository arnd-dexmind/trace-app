COMPONENT TREE — OPERATOR CONSOLE
============================================================

Prototype: src/design/prototypes/operator-console.html

ConsoleLayout
  Grid: sidebar(280px) | main(1fr) | actions(320px)
  On <=1024px: single-column with fixed actions overlay
  On <=640px: queue panel max-height 40vh
  ---
  Props: none (page-level layout)
  States: default, actions-open (mobile only)

  ConsoleHeader
    slot:left    — logo + space badge
    slot:right   — walkthrough selector + batch-accept
    ---
    Props: spaceName, walkthroughOptions[], pendingBatchCount

    WalkthroughSelect
      Select for active walkthrough.
      Props: options: {id, label}[], selected, onChange

    BatchAcceptButton
      Only visible when pendingBatchCount > 0.
      Props: count, onClick

  QueuePanel
    Left sidebar. Three tabs.
    ---
    Props: pending[], inReview[], completed[]
    States: tab=pending | inReview | completed

    QueueTab
      Props: label, count, selected, onClick
      States: default, selected

    CandidateCard
      Queue item. Click selects it in the main panel.
      Props: item, confidence, location, repairSummary, active
      States: default, active
      Sub-components: ConfidenceBadge

      ConfidenceBadge
        Props: score (0–100)
        States: high (>=85%, green), medium (60–84%, yellow),
                low (<60%, red)
        Renders dot + percentage.

  MainPanel
    Center area. Shows selected candidate or empty state.
    ---
    States: empty (no candidate selected), detail (candidate selected)

    EmptyState
      Props: icon, message
      Used when no candidate is selected.

    CandidateDetail
      Props: candidate { id, itemName, category, proposedLocation,
              zone, storagePath, frameTimestamp, repairCategory,
              repairSeverity, confidence, priorConfirmations,
              locationHistory[], evidenceUrl }

      EvidenceFrame
        Props: imageUrl (or placeholder), frameNumber, zone
        States: loaded, placeholder (no image available)
        Aspect ratio: 16/9. Overlay badges at bottom.

      DetailGrid
        2-column grid (stacks to 1-col on mobile).
        Each cell: label + value + optional sub-value.

      LocationTimeline
        Props: entries[] { date, location, note, tags[] }
        Vertical timeline with dots.
        Dot states: current (brand), moved (warning),
                    resolved (success), default (neutral)
        Tag sub-component: TagBadge (kind: confirmed|relocated|repair)

  ActionsPanel
    Right sidebar. Fixed overlay on mobile with toggle button.
    ---
    Props: candidate (same as CandidateDetail)
    States: default, open (mobile)

    QuickAccept
      Only shown when confidence >= 90%.
      Green callout with single confirm button.
      Props: onConfirm

    IdentityActions
      Relabel select + Confirm/Mark Unseen buttons.
      Props: itemOptions[], onConfirm, onMarkUnseen

    LocationActions
      Location correction select + Update button.
      Props: locationOptions[], onUpdate

    RepairActions
      Three repair decisions: Accept/Downgrade/Reject.
      Props: onAccept, onDowngrade, onReject

    MergeActions
      Merge-with select + Merge button.
      Props: mergeOptions[], onMerge

    ResolutionNote
      Textarea + Mark Resolved button.
      Props: onResolve

    NavigationActions
      Previous / Skip & Next buttons.
      Props: onPrevious, onSkipNext

  ActionsToggle (mobile only)
    FAB button to open/close actions panel.
    Props: open, onToggle

  ActionsOverlay (mobile only)
    Backdrop when actions panel is open.
    Props: open, onClick (closes panel)

COMPONENT TREE — UPLOAD FLOW
============================================================

Prototype: src/design/prototypes/upload.html

  UploadPage
    Shell: max-width 640px, centered.

    TopNav (shared)

    SpaceSelector
      Select dropdown that chooses target space.
      Props: spaces[], selected, onChange

    UploadFlow (state machine)
      States: idle → selected → processing → complete | error

      IdleState
        DropZone with icon, title, hint, format info.
        Also renders hidden <input type="file">.

        DropZone
          Props: onFileSelect
          States: default, drag-over
          Handles: click to browse, dragover/dragleave/drop events.
          Accepts: video/mp4, video/quicktime, application/zip.

      SelectedState
        Shows selected file info + Upload button.

        SelectedFile
          Props: name, size, onRemove
          File icon, name (truncated), size, remove button.

      ProcessingState
        ProgressBar + stage list.

        ProgressBar
          Props: percent, label
          Track + fill. Fill animates via CSS transition.

        ProcessingStages
          List of StageItem components.
          Stages: Upload video, Extract frames, AI item detection,
                  Review queue ready.

          StageItem
            Props: label, state (pending|active|done)
            Icon states: pending (empty circle), active (pulsing brand),
                         done (green check)

      CompleteState
        ResultCard (success variant) with stats and CTA buttons.
        Stats: items detected, pending review, repairs flagged.
        CTAs: Open in Operator Console, Upload Another.

      ErrorState
        ResultCard (error variant) with message + Try Again button.

IMPLEMENTATION NOTES
============================================================

1. FRAMEWORK
   - Components are framework-agnostic. The prototypes are vanilla HTML/CSS.
   - If adopting React: each section under COMPONENT TREE maps to a
     component file (e.g., ConsoleLayout.tsx, QueuePanel.tsx).
   - If staying server-rendered: use the same CSS token system with
     template partials matching the component breakdown above.

2. MOBILE-FIRST
   - All prototypes use min-width media queries (mobile-first).
   - Touch targets are minimum 44px (--sm-touch-target).
   - Operator console collapses to single-column on tablet/mobile
     with a fixed overlay for the actions panel.

3. ACCESSIBILITY
   - Focus indicators on all interactive elements (btn:focus-visible).
   - ARIA labels on icon-only buttons (aria-label).
   - Role attributes on composite widgets (tablist, tab, combobox, listbox).
   - Skip link recommended for operator console.
   - Color is never the sole indicator — all badges include text labels.

4. AUDIT TRAIL
   - Every operator action records: timestamp, operator ID, action type,
     previous value, new value, evidence note.
   - LocationTimeline entries link to the specific walkthrough + frame
     that produced the location observation.

5. DATA CONTRACTS
   - Candidate: { id, itemId, itemName, category, proposedLocation,
     zone, storagePath, frameNumber, frameTimestamp, repairCategory,
     repairSeverity, confidence, priorConfirmations, locationHistory,
     evidenceUrl }
   - Item: { id, name, category, status, lastSeenDate, lastSeenContext,
     firstCataloguedDate, zone, position, storagePath }
   - Repair: { id, itemId, itemName, title, status, severity,
     location, detectedDate, resolvedDate }
   - LocationEntry: { date, location, note, tags: ('confirmed'|'relocated'|'repair'|'catalogued')[] }
